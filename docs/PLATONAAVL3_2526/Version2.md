Dans votre projet GitHub il faut :

un readme.md qui explique les objectifs du projet (reprenez ce qui existe déjà et expliquer ce que votre groupe apporte en plus).

Quelles sont les json fournis par votre partie et comment les interprété: je vous demande un peu d'imagination ici sur l'usage que l'on peux faire de votre travail dans le contexte de platon : imaginez les usage de ce que votre groupe a fournis.



un install.md qui explique comment installer le projet une fois cloné su une machine, fichier requirements.txt environnement etc.
