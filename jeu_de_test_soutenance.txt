JEU DE TEST SOUTENANCE
Projet Python Option Groupe 5

Ce fichier sert de fil conducteur pendant la soutenance.
Le but est de montrer un parcours simple, propre et qui marche vraiment avec nos donnees.

Ce qu'il faut lancer avant de passer

1. Lancer l'API dans le dossier du projet

uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

2. Verifier vite fait que l'API tourne sur

http://127.0.0.1:8000/docs

3. Ouvrir un autre terminal

4. Aller dans le dossier du client lourd

cd client_lourd

5. Lancer le client

flet run main.py

Ce qu'on peut dire au debut

Notre application est composee d'une API FastAPI et d'un client lourd Flet.

Le but est de gerer des AAV, afficher leur detail et suivre la progression d'un apprenant.

On a aussi mis en place une partie phase 3 pour simuler une tentative et voir l'effet sur le suivi.

Parcours conseille pendant la soutenance

1. Montrer que l'API tourne
2. Montrer la liste des AAV
3. Cliquer sur un AAV pour voir le detail
4. Montrer la modification des prerequis
5. Charger un apprenant
6. Montrer le suivi apprenant
7. Simuler une tentative
8. Montrer que la progression change

Test 1
Verifier que l'API fonctionne

Action

- montrer le terminal avec le serveur
- montrer eventuellement la page /docs

Ce qu'on doit voir

- l'API se lance sans erreur
- les routes sont accessibles

Ce qu'on peut dire

On a bien une API fonctionnelle avec les routes principales des AAV, des apprenants et des tentatives.

Test 2
Afficher la liste des AAV

Action

- ouvrir le client lourd
- cliquer sur Appliquer si la liste n'apparait pas tout de suite

Ce qu'on doit voir

- la liste des AAV a gauche
- pour chaque AAV on voit son id, son nom, sa discipline et son type

Exemple d'AAV qu'on peut montrer

- AAV 1 : Types entiers
- AAV 2 : Type caractère
- AAV 5 : Opérateurs arithmétiques

Ce qu'on peut dire

Ici on charge les AAV depuis l'API et on les affiche dans le client lourd.

Test 3
Filtrer les AAV

Action

- ecrire Programmation dans le champ Discipline
- ou choisir Atomique dans le menu Type
- cliquer sur Appliquer

Ce qu'on doit voir

- la liste se met a jour

Ce qu'on peut dire

On peut filtrer les AAV pour retrouver plus facilement ce qu'on cherche.

Test 4
Afficher le detail d'un AAV

Action

- cliquer sur AAV 1 Types entiers

Ce qu'on doit voir a droite

- ID : 1
- nom : Types entiers
- type : Atomique
- exercices lies : 101, 102
- prerequis
- description

Autres exemples valides

- AAV 2 Type caractère avec exercices 103, 104
- AAV 5 Opérateurs arithmétiques avec exercices 109, 110

Ce qu'on peut dire

Quand on clique sur un AAV, on affiche son detail dans le panneau de droite.

On a aussi ajoute les exercices lies a l'AAV pour avoir un affichage plus complet.

Test 5
Modifier les prerequis

Action

- selectionner un AAV
- cliquer sur Modifier prerequis
- changer la liste des ids
- enregistrer

Ce qu'on doit voir

- l'enregistrement se fait sans erreur
- les prerequis se mettent a jour dans le detail

Ce qu'on peut dire

On peut modifier les prerequis directement depuis l'interface, et la mise a jour passe ensuite par l'API.

Test 6
Charger un apprenant

Action

- aller dans la zone Phase 3 Suivi apprenant
- choisir l'apprenant 2 bob_progressif
- cliquer sur Charger apprenant

Ce qu'on doit voir

- un resume apparait
- le nombre total d'AAV suivis
- les AAV maitrises
- les AAV en cours
- les AAV non commences
- le taux global
- le detail des AAV suivis

Pourquoi prendre bob_progressif

Cet apprenant est pratique pour la demo car il a deja plusieurs AAV en cours, donc on voit bien l'evolution apres une tentative.

Ce qu'on peut dire

Cette partie permet d'afficher le suivi d'un apprenant a partir des donnees du backend.

Test 7
Lire le detail du suivi apprenant

Action

- regarder les lignes ou cartes des AAV suivis pour bob_progressif

Ce qu'on peut voir

- AAV 1 avec un niveau deja maitrise
- AAV 5 avec un niveau intermediaire
- AAV 6 avec un niveau plus faible

Ce qu'on peut dire

Le but ici est de voir rapidement ou en est l'apprenant sur chaque AAV.

Test 8
Simuler une tentative

Action conseillee pour un exemple qui marche bien

- apprenant : 2 bob_progressif
- ID AAV cible : 6
- ID exercice : 111
- score : 0.8

Autre exemple valide

- apprenant : 2 bob_progressif
- ID AAV cible : 5
- ID exercice : 109
- score : 0.8

Ce qu'on doit voir

- la tentative est creee
- la tentative est traitee
- un message de succes apparait
- le suivi apprenant se recharge
- le niveau de maitrise augmente

Exemple de resultat attendu sur le cas AAV 6

- avant la tentative, le niveau etait autour de 0.3
- apres la tentative, le niveau monte autour de 0.55

Ce qu'on peut dire

Ici on simule une tentative sur un AAV pour mettre a jour la progression de l'apprenant.

On cree d'abord la tentative puis on la traite pour recalculer le niveau de maitrise.

Test 9
Montrer que la progression change

Action

- comparer le suivi juste avant et juste apres la simulation

Ce qu'on doit voir

- une augmentation du niveau de maitrise
- un historique de tentative qui s'allonge
- parfois un changement d'etat si le seuil est atteint

Ce qu'on peut dire

Ca montre que l'application ne fait pas juste de l'affichage, il y a bien une mise a jour des donnees apres une action utilisateur.

Si le prof demande ce qu'on a fait

Reponse possible

On a travaille sur l'integration globale de l'application, l'affichage des AAV, leur detail, le suivi apprenant et la simulation de tentative.

On a surtout voulu avoir une application coherente, utilisable et presentable en client lourd.

On a relie la base, l'API et l'interface pour que tout fonctionne ensemble.

Si quelque chose bug pendant la demo

Verifier rapidement

- que l'API tourne bien
- que le client est bien lance
- que la base de donnees est presente
- que l'ID d'AAV existe bien
- que le score est entre 0 et 1

Phrase simple a dire

On verifie juste le lancement des services, mais normalement le parcours principal fonctionne.

Conclusion simple

Le projet permet de gerer les AAV, de consulter leur detail et de suivre un apprenant avec mise a jour apres tentative.

On a voulu faire une application claire, simple a utiliser et suffisante pour montrer un vrai fonctionnement pendant la demo.
